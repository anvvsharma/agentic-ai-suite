# Learning Path

🎓 Complete structured learning path for GenAI & Agentic AI mastery.

## Overview

This directory contains a comprehensive, hands-on learning path covering 13 phases with 70+ subtopics. The structure is designed to take you from prerequisites to advanced AI agent development, with each phase building upon the previous one.

## Learning Path Structure

### Phase 1: Prerequisites & Foundations
**Directory**: `01-prerequisites/`

Essential skills and tools before diving into AI development.

**Topics**:
- `async-programming/` - Asynchronous programming patterns
- `data-engineering-basics/` - Data processing fundamentals
- `pydantic/` - Data validation and modeling
- `python-advanced/` - Advanced Python concepts

**Duration**: 1-2 weeks  
**Prerequisites**: Basic Python knowledge

---

### Phase 2: LLM Fundamentals
**Directory**: `02-llm-fundamentals/`

- `prompt-engineering/` - Crafting effective prompts

## Learning Approach

### 1. Sequential Learning
Follow phases in order. Each phase builds on previous knowledge.

### 2. Hands-On Practice
- Read theory and documentation
- Implement examples in each subtopic folder
- Experiment with variations
- Document your learnings

### 3. Project-Based Learning
- Apply concepts in `projects/` directory
- Build small POCs for each phase
- Combine multiple concepts
- Move polished work to `finalized-pocs/`

### 4. Documentation
- Keep notes in each subtopic folder
- Document challenges and solutions
- Create comparison notes
- Write blog posts in `blog-activity` repo

## Getting Started

1. **Set Up Environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **Configure API Keys**
   Create `.env` file:
   ```env
   OPENAI_API_KEY=your_key
   GROQ_API_KEY=your_key
   # Add other keys as needed
   ```

3. **Start with Phase 1**
   Navigate to `01-prerequisites/` and begin learning

4. **Track Progress**
   - Use checkboxes in this README
   - Keep a learning journal
   - Document in blog posts
